## 一、开发总则

### 1.1 先 API 后框架

> Anthropic: "developers start by using LLM APIs directly: many patterns can be implemented in a few lines of code."

**开发铁律：所有组件的第一版必须直接调用 LLM API 实现。确认可行、评估通过后，才可考虑引入框架。**

| 禁止行为 | 理由 |
|---------|------|
| MVP 阶段引入 LangChain/LlamaIndex/CrewAI | 框架抽象层遮蔽 prompt 和 response，增加调试难度 |
| 因为"框架有现成的"跳过直接 API 验证 | 最成功的 Agent 实现不依赖复杂框架 |
| 在框架上再套自研抽象 | 除非有明确的多个框架切换需求 |

## 二、组件 1：共享记忆系统

### 2.1 核心接口：MemoryGateway

```python
class MemoryGateway:
    """记忆系统统一入口。所有 Agent 通过此接口读写记忆。"""

    async def write(self, entry: MemoryEntry, *, trace_id: str) -> str:
        """写入记忆。内部走 Write Policy 评估 → 去重 → ChromaDB + SQLite。"""

    async def read(self, entry_id: str, *, agent_id: str, trace_id: str) -> Optional[MemoryEntry]:
        """按 ID 读取。自动更新 access_count 和 last_accessed_at。"""

    async def search(self, query: MemoryQuery, *, trace_id: str) -> list[MemorySearchResult]:
        """多路检索（语义 + 关键词 + 标签）→ 合并去重 → LLM 重排序 → top_k。"""

    async def consolidate(self, *, agent_id: str, session_id: str, trace_id: str) -> list[str]:
        """工作记忆中 importance ≥ 4 的条目提升到长期记忆。"""

    async def decay(self, *, layer: str, trace_id: str) -> int:
        """运行遗忘策略，返回淘汰数。"""

    async def share(self, entry_id: str, from_agent: str, to_agents: list[str], *, trace_id: str) -> bool:
        """跨 Agent 共享记忆。"""
```

### 2.2 长期记忆 SQLite DDL

```sql
CREATE TABLE memories (
    id TEXT PRIMARY KEY,
    agent_id TEXT NOT NULL,
    content TEXT NOT NULL,              -- ≤ 2000 字符
    memory_type TEXT NOT NULL,          -- fact/decision/experience/preference/task_context
    importance REAL NOT NULL DEFAULT 0.5,
    tags TEXT,                          -- JSON array
    source_trace_id TEXT,
    created_at TEXT NOT NULL,
    last_accessed_at TEXT NOT NULL,
    access_count INTEGER DEFAULT 0,
    expires_at TEXT,                    -- NULL = 永不过期
    chroma_id TEXT UNIQUE
);
CREATE INDEX idx_memories_agent ON memories(agent_id);
CREATE INDEX idx_memories_type ON memories(memory_type);
CREATE INDEX idx_memories_importance ON memories(importance DESC);
```

### 3.2 引擎核心

```python
class WorkflowEngine:
    async def execute(self, workflow: WorkflowDefinition, inputs: dict, *, trace_id: str) -> WorkflowResult:
        """
        1. 校验工作流定义
        2. PreWorkflow hook
        3. 按拓扑排序执行 steps（每个 step 前后触发 PreStep/PostStep hooks）
        4. 异常按 retry 配置处理，全部失败标记 partial_success
        5. PostWorkflow hook
        6. 返回完整结果 + trace
        """

    async def resume(self, run_id: str, *, from_step: str, trace_id: str) -> WorkflowResult:
        """从指定步骤恢复（人工审批后继续）。"""
```

### 6.1 Hooks 机制

```python
class HookManager:
    async def pre_tool_use(self, agent_id, tool_name, params, *, trace_id) -> HookDecision:
        """检查：权限 / 预算 / 频率限制 / 参数安全性 → allow | block | escalate"""

    async def post_tool_use(self, agent_id, tool_name, result, *, trace_id) -> HookDecision:
        """检查：结果含敏感信息 / 异常大 / 含指令注入 → allow | block | escalate"""
```

## 八、Prompt 版本管理

### 8.1 版本规则

```
v{major}.{minor}
major: 不兼容变更（改输出格式、工具调用方式、角色定义）
minor: 兼容变更（优化措辞、补充示例、调整默认值）
```

### 8.2 变更流程

```
修改 Prompt → 版本号 +1 → 跑离线评估 → 对比基线
  ├─ 提升 → 合并，更新基线
  ├─ 持平 → 合并，记录原因
  └─ 下降 → 阻断 → 分析 → 修改 → 重试
```

## 十、代码约定

| 规则 | 说明 |
|------|------|
| 类型注解 | 所有公共函数完整类型签名，mypy strict mode 零错误 |
| 异步 | I/O 用 async/await，CPU 密集用 run_in_executor |
| 错误处理 | 自定义异常层级：AgentSystemError → MemoryError/WorkflowError/... |
| 日志 | structlog，每条日志带 agent_id 和 trace_id |
| 测试 | pytest + pytest-asyncio，mock LLM，real ChromaDB (tmp path) |
| 配置 | 所有可变参数通过环境变量注入，零硬编码 |
| 行数 | 每个模块 ≤ 500 行（Karpathy 原则），超出则拆分 |
| MCP 优先 | 新工具必须通过 MCP 暴露，Agent 不直接调用 Python 函数 |
