## 二、Phase 0: 评估基建（第 1 周）

**在写代码前先建立评估基准。依据 LangChain State of Agent Engineering: quality is the #1 production killer。**

| 周次 | 任务 | 输出 | Gate 检查点 |
|------|------|------|------------|
| W1-D1/2 | 定义每个 Agent 的任务成功标准（准确率目标、延迟 SLA、成本上限） | 成功标准矩阵 | 所有 11 Agent 的成功标准量化完毕 |
| W1-D2/3 | 编写 Happy Path 测试用例（每个 Agent 3-5 条） | Happy Path 测试集 | 至少 33 条用例，覆盖所有 Agent |
| W1-D3/4 | 编写 Edge Cases（异常输入、工具失败、并发冲突） | Edge Cases 测试集 | 覆盖 problems 文档 Top 10 高频问题 |
| W1-D4/5 | 编写 Adversarial 测试（Prompt 注入、越权操作、幻觉诱导） | Adversarial 测试集 | 覆盖 problems 5.1/5.3 章节 |
| W1-D5 | 搭建评估脚本框架（离线评估 + 回归测试） | 评估 CLI | `run-eval` 命令可执行且产出评分报告 |

**W1 退出标准：** 测试数据集就绪 + 评估框架跑通（即使 Agent 还没改）。

> 评估先行：LangChain "offline evaluations on test sets" 是生产前提。

## 三、Phase 1: 原型（第 2-4 周）

**目标：6 核心组件最小可行实现，单 Agent 全流程验证。**

### Week 2: 共享记忆 + 经验学习（地基）

| 天 | 任务 | 输出 |
|----|------|------|
| W2-D1 | 设计记忆架构（短期/长期/工作记忆 schema） | 记忆 schema 文档 |
| W2-D2 | 实现 ChromaDB 长期记忆存储 + CRUD API | `memory_store.py` |
| W2-D3 | 实现经验学习自动提取（任务失败/成功触发） | `experience_extractor.py` |
| W2-D4 | 实现 Agent 启动时自动注入相关记忆/经验 | `memory_injector.py` |
| W2-D5 | 集成 researcher Agent 验证记忆读取/写入闭环 | researcher 端到端通过 |

**Gate: researcher Agent 执行 5 个同类任务，第 3-5 次使用时检索到前两次的经验。**

### Week 3: 安全护栏 + 审批系统（保护层）

| 天 | 任务 | 输出 |
|----|------|------|
| W3-D1 | 定义操作风险分级（L0-L3）+ 每级审批规则 | 风险矩阵文档 |
| W3-D2 | 实现工具调用拦截（PreToolUse hook） | `safety_guard.py` |
| W3-D3 | 实现飞书审批卡片（L2/L3 操作需确认） | 审批 bot |
| W3-D4 | 实现活动日志（JSONL，每条操作一行） | `activity_logger.py` |
| W3-D5 | 实现成本追踪（按 Agent/任务归因 token 消耗） | 成本报表 |

**Gate: tech-dev Agent 执行高风险操作（写文件/调 API）时触发审批，日志完整记录。**

### Week 4: 可观测性 + 评估闭环 + 整合测试

| 天 | 任务 | 输出 |
|----|------|------|
| W4-D1 | 实现 Agent 标识符注入（x-agent-id header） | 全链路 trace |
| W4-D2 | 实现飞书每日监控面板（活跃度/消耗/异常） | 监控 bot |
| W4-D3 | 实现任务执行后自动评估（自评评分卡） | `auto_eval.py` |
| W4-D4 | 集成测试：researcher → tech-dev 流水线 + 记忆共享 | 集成测试报告 |
| W4-D5 | 阿禹验收 Phase 1：所有组件跑通的 demo 演示 | 验收通过/问题清单 |

**Phase 1 退出标准：**
- 6 核心组件全部实现且通过单元测试
- researcher + tech-dev 端到端流水线跑通
- 共享记忆读写延迟 < 500ms
- 经验学习闭环验证（重试同类任务时自动检索历史经验）
- 审批系统对 L2/L3 操作拦截成功率 100%

**目标：全部 11 Agent 上线，安全护栏和可观测性覆盖所有 Agent。**

| 周次 | 任务 | Gate |
|------|------|------|

**Phase 2 退出标准：**
- 所有 11 Agent 部署新的记忆/经验/安全系统
- 2 条以上工作流流水线正常运行
- 监控面板每日自动推送
- Prompt 注入防御覆盖率 100%
- 无已知高危安全漏洞

## 六、风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| LLM 幻觉导致经验提取错误 | 高 | 中 | 经验条目需人工确认后写入（Phase 1 默认阿禹审批） |
| 多 Agent 协作死锁 | 中 | 高 | 每个 relay 调用设 120s timeout + 降级为人工介入 |
| ChromaDB 大规模检索性能下降 | 低 | 中 | Phase 1 限制记忆条目 < 1000，Phase 2 加入分片 |
| Token 成本失控 | 中 | 高 | 每日成本预警（超过预设上限自动降级） |
| Prompt 注入绕过安全护栏 | 中 | 高 | 多层防御：输入 sanitize + 工具权限边界 + 审批 |
